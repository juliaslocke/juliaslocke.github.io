When looking at my code in Safari, Firefox and Chrome, I notice a difference in the form. In Chrome and Firefox, the DOB field has mm/dd/yyyy to tell users how the data should be input. In Safari, the DOB must be typed in, in Chrome, an arrow can be used to change the value of the month, date, or year depending on which is focused on, and in Firefox, a calendar appears to allow DOB selection. Along with this the audio and video controls are different between the three browsers. Currently, the form only allows you to select an option if you click directly on the radio button. Changing this to allow users to click on the labels as well would make the form easier to use because there will be a larger target. This can be done by wrapping the input tag in a label tag. The radio buttons also are not separated by snacks and drinks. You cannot select both a snack and a drink. Changing the name of the drink items from "snack" to "drink" will fix this. After hosting my code, I did not find any new issues, but I have a remaining warning about an incompatibility across broswers with the date input.
